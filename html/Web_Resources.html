Top 5 Resources

Each entry includes what it offers, who it helps, how I’ll use it, and a usefulness rating.

1) MDN Web Docs (Mozilla)

Link: https://developer.mozilla.org/

What it offers: The most practical reference for HTML, CSS, and JavaScript. Clear definitions, examples, browser support notes, and deep dives on core web APIs.

Who it’s useful for: Everyone especially when you need correct syntax fast.

How I’ll use it:

  • Quickly verify HTML/CSS properties and correct usage.
  • Learn web APIs for interactive pages.
  • Check browser compatibility before using newer features.

Usefulness: 10/10

2) web.dev (Google)

Link: https://web.dev/

What it offers: Best-practice guidance for modern web development: performance, accessibility, SEO, responsive design, and real-world patterns.

Who it’s useful for: Developers who want their sites to load fast, rank better, and work well on mobile.

How I’ll use it:

  • Improve page speed and responsiveness.
  • Use Lighthouse guidance to find and fix issues.
  • Adopt accessibility patterns so my sites work for more users.

Usefulness: 9/10

3) CSS-Tricks (Front-end patterns)

Link: https://css-tricks.com/

What it offers: Practical CSS and layout recipes: Flexbox/Grid patterns, navigation bars, card layouts, responsive techniques, and UI problem-solving examples.

Who it’s useful for: Anyone who can “write CSS” but wants to make pages look clean, consistent, and readable without fighting layout bugs.

How I’ll use it:

  • Build nicer layouts faster.
  • Fix common UI issues: alignment, spacing, overflow.
  • Learn patterns I can reuse across multiple assignments.

Usefulness: 8.5/10

4) GitHub Docs (Pages + Git)

Link: https://docs.github.com/

What it offers: Official documentation for GitHub features especially GitHub Pages publishing, repositories, branches, pull requests, and workflows.

Who it’s useful for: Anyone publishing class work to the internet and managing versions cleanly.

How I’ll use it:

  • Correctly configure GitHub Pages / gh-pages publishing.
  • Use git workflows that prevent “broken deployment” mistakes.
  • Understand repo settings, branches, and deployment behavior.

Usefulness: 9/10

5) Supabase Docs (Cloud database + auth)

Link: https://supabase.com/docs

What it offers: A cloud backend toolkit. Great for turning a static site into a real app without building everything from scratch.

Who it’s useful for: Full-stack developers and students who want to ship projects that store data, handle logins, and behave like real products.

How I’ll use it:

  • Prototype full-stack apps with real data.
  • Add authentication for user accounts in portfolio projects.
  • Use hosted services to learn practical cloud patterns.

Usefulness: 8.5/10

What I discovered (quick summary)

  • MDN is the fastest way to verify correct HTML/CSS/JS and understand browser behavior.
  • web.dev helps turn “it works” into “it works well”.
  • CSS pattern libraries save time and reduce layout trial-and-error.
  • GitHub Pages is straightforward, but only if your links and paths are correct.
  • Cloud services like Supabase can quickly level up projects into real apps with data + auth.
